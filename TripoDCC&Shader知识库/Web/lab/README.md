## 为什么代码能塞进 Obsidian 库

前端项目通常有个 `node_modules/`（几万个文件的依赖目录），会**拖垮 Obsidian 索引/同步、污染 git**。本 lab 用两招绕开：

1. **前 5 级零构建（zero-build）**：只用纯 `.html` / `.css` / `.js` 文件，浏览器直接打开就能跑；需要库（如 Three.js）时走 **CDN + import maps**，不下载依赖、没有 `node_modules`。
   - 类比：像你直接 `#include` 一个 header、或引擎里拖一个 `.dll`——不需要包管理器。
2. **真·工具链留到后面单独一课**：等你想学 Vite/pnpm 时才引入 `node_modules`，且已在 `.gitignore` 和 Obsidian「忽略文件」里屏蔽，不会污染库。

> 结论：前几级你**双击 `index.html` 就能看到结果**（或用下面的本地服务器），零安装、零配置。

## 怎么运行

- **最简单**：文件管理器里双击 `index.html`，浏览器打开即可（纯静态页足够）。
- **需要 fetch/模块时**（L3 起，`file://` 会被浏览器安全策略拦）：在该级目录起一个本地服务器——
  ```bash
  # 任选其一，都不装依赖
  npx serve .        # 有 node 就能用
  python3 -m http.server 8000
  ```
  然后浏览器开 `http://localhost:8000`。
- 改完代码**刷新浏览器**即可，无需重新构建。
