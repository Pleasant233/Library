# L1 · 静态个人名片

> Lab 第一关。目标：**第一次亲手把 HTML 结构 + CSS 样式拼出一个能看的页面**，建立「结构 vs 样式」的肌肉记忆。
> 理论对照：[[01 HTML与CSS]]、[[00.3 CSS上手]]；在阶梯里的位置：[[11 实战项目路线（AI协作版）]] L1。
> 运行方式见 [[README]]（本关双击 `index.html` 即可）。

---

## 🎯 做出什么

一张个人/角色名片：头像圆形、名字、一句签名、几个技能标签、一个按钮。纯静态，不需要 JS。

产出文件（就在本文件夹旁边）：
- `index.html` —— 页面**结构**（有哪些元素、谁套谁）。
- `style.css` —— 页面**样式**（长什么样：颜色、间距、圆角、布局）。

> 引擎类比：`index.html` 像**场景 Hierarchy**（有哪些物体、父子关系），`style.css` 像**材质 + Transform**（每个物体长啥样、怎么摆）。

---

## 🧠 必须自己会的范式（这一关的重点：练「读 / 判」）

不要求手打，但下面这些你**读代码时必须认得**，否则后面全是黑话：

1. **标签/元素**：`<h1>名字</h1>` = 开标签 + 内容 + 闭标签，合起来是一个「元素」。
2. **嵌套 = 父子**：`<div><img><h1></h1></div>`，缩进就是 Hierarchy 层级 → 浏览器里叫 **DOM 树**。
3. **class 是"牌子"**：`<div class="card">` 给元素挂个牌子，CSS 用 `.card { }` 按牌子找到它上样式。
4. **盒模型**：每个元素都是一个盒子 = `content + padding + border + margin`。`padding` 撑内部、`margin` 推外部。
5. **CSS 挂载方式**：本关用**外链** `<link rel="stylesheet" href="style.css">`（结构与样式分离，最规范）。

> 检验「你真读懂了」的判据（[[学习路径设计方法论]] 原则 4）：**能不能改一处、并预测结果？** 见下方验收。

## 🤖 怎么驱动 AI 写这一关

代码交给 AI（本关由 Codex 生成）。你要会的是**把需求讲清**（上下文工程）。参考指令：

> 「用纯 HTML+CSS 写一张个人名片，不要 JS、不要任何框架、不要 node 依赖，只输出 `index.html` 和 `style.css` 两个文件。要求：圆形头像、名字、签名、3 个技能标签、1 个按钮；用 flex 居中，卡片有圆角和阴影；CSS 走外链。代码要有中文注释解释盒模型和 flex。」

拿到代码后，你的活是**读 / 判 / 查**：读懂每段在干嘛、判断是否符合要求、跑起来看效果。

## ✅ 验收（改一处，预测结果，再跑验证）

双击 `index.html` 打开，然后**逐条动手**，先在脑子里预测、再改、再看对不对：

- [ ] 把 `.card` 的 `padding` 从 `24px` 改成 `48px` → 预测：卡片内部更空、整体变大。
- [ ] 把某个技能标签的 `background` 改个颜色 → 预测：只有那类标签变色。
- [ ] 把 `.card` 的 `border-radius` 改成 `0` → 预测：圆角消失变方块。
- [ ] 把头像 `img` 的 `border-radius: 50%` 删掉 → 预测：头像从圆形变方形。
- [ ] 把容器的 `justify-content: center` 删掉 → 预测：卡片不再水平居中，靠左。

**5 条预测全对 = 你真的懂了结构与样式的对应关系**，可以进 L2。有对不上的，回看 [[00.3 CSS上手]] 对应段落补上。
