# 00.3 CSS 上手

> 承接 [[00 前端是什么]]「CSS 管样式」。你对 HTML/CSS 完全零基础，所以这篇**先花一节把"标签、元素、div、class、样式"这些字面黑话拆开**，再讲 CSS 的机制。
> 说明：CSS 在你的 C++ / C# / HLSL 背景里**几乎没有对应经验**——它不是编程，而是一套「声明式排版规则」。硬要类比，它像引擎里调材质/后处理：你不写循环，而是**声明一堆属性**，由渲染器（这里是浏览器排版引擎）去出结果。
> 看完再进 [[01 HTML与CSS]]（HTML 完整骨架 + Flex/Grid/响应式/架构）。

## 零、先补 4 个最基础的词（黑话拆解）

进 CSS 之前，你得先能看懂它操作的对象是什么。下面每个词都从字面拆。

### 1. 标签（tag）与元素（element）：网页内容的最小积木

HTML 里的内容，是用一对**尖括号包起来的"标签"**写的。最典型的一段：

```html
<h1>你好</h1>
```

逐字拆解：
- `<h1>` —— **开始标签（opening tag）**。`h1` 是标签名（heading level 1，一级标题）。尖括号 `< >` 是 HTML 的语法符号，念"尖括号"。
- `你好` —— **内容**，真正显示在屏幕上的文字。
- `</h1>` —— **结束标签（closing tag）**，多了个斜杠 `/`，表示"这个标签到此为止"。
- **整个 `<h1>你好</h1>` 合起来，叫一个"元素（element）"**。

> 一句话：**标签是写法（那对尖括号），元素是它构成的那个"东西"**。日常口语里两个词经常混用，不用纠结。

有些元素没有内容、不需要结束标签，叫**自闭合标签**，比如图片：
```html
<img src="cat.png">   <!-- 显示一张图，src 指定图片地址，没有 </img> -->
```

常见标签速查（详见 [[01 HTML与CSS]]）：
| 标签 | 是什么 |
| --- | --- |
| `<h1>`~`<h6>` | 标题，1 最大 6 最小 |
| `<p>` | 段落（paragraph） |
| `<a>` | 链接（anchor），点了跳转 |
| `<img>` | 图片 |
| `<button>` | 按钮 |
| `<span>` | **无语义的行内容器**（包一小段文字用） |
| `<div>` | **无语义的块级容器**（见下） |

### 2. div 到底是什么

`<div>` 是出现频率最高、也最让新手困惑的标签。它的真相很简单：

**`<div>` 就是一个"什么含义都没有的空盒子"，纯粹用来把一堆内容框在一起、方便统一排版或统一上样式。**

> 类比你的引擎：`<div>` ≈ Hierarchy 里一个**空的 GameObject**。空物体本身不渲染任何东西，你建它只是为了当**容器/分组**——把几个子物体挂在它下面，一起移动、一起管理。`<div>` 完全一样：它自己不显示成任何可见样子，你用它把标题、图片、按钮包成"一张卡片"，再对这个盒子整体设背景、圆角、间距。

```html
<div>                    <!-- 一个盒子，把下面三样框成"一张卡片" -->
  <img src="cat.png">
  <h2>小猫</h2>
  <button>购买</button>
</div>
```

`<span>` 是 `<div>` 的"行内版"——同样无含义，但用于包**一行文字里的一小段**（比如把一句话里的某个词标红）。两者区别（block vs inline）第三节讲。

### 3. 属性（attribute）与 class / id：给元素贴的"标签牌"

标签的尖括号里，除了标签名，还能写**属性**，格式是 `名字="值"`：

```html
<img src="cat.png" alt="一只猫" width="200">
     └─属性─┘      └──属性──┘   └─属性─┘
```

- `src`、`alt`、`width` 都是**属性名**，`="..."` 里是**属性值**。
- 最重要的两个通用属性是 **`class`** 和 **`id`**，它们是给元素贴的"牌子"，好让 CSS/JS **找到**这个元素：

```html
<div class="card">...</div>     <!-- 给这个 div 贴一张名为 card 的牌 -->
<div id="header">...</div>      <!-- 给这个 div 一个唯一编号 header -->
```

- **`class`（类）**：一个可**复用**的标记，很多元素可以贴同一个 `class`，用来"批量选中一类元素统一上样式"。
  > 类比：像 Unity 的 **Tag**，或者"这批物体都用同一个材质"——`class="card"` 相当于说"这些都是卡片，套卡片样式"。一个元素还能贴多个 class：`class="btn primary"`。
- **`id`**：全页面**唯一**的编号，一个 id 只能有一个元素用。用于指代那个独一无二的元素。

**这就是 `div` 和 `class` 的关系**：`div` 是盒子本身，`class="card"` 是贴在盒子上的牌子——CSS 待会儿就靠这张牌子找到它、给它上样式。

### 4. "样式（style）"是什么，CSS 是什么

**"样式"就是一个元素长什么样的所有视觉属性**：颜色、字号、宽高、间距、边框、圆角、背景、位置、阴影、动画……

> 类比：一个元素的"样式" ≈ 你给一个物体配的**材质参数 + Transform 排版**。HTML 决定"有哪些物体、谁套谁"，样式决定"每个物体长啥样、摆哪"。

**CSS（Cascading Style Sheets，层叠样式表）就是"专门写样式的语言/文件"**。一条 CSS 规则长这样：

```css
h1 {              /* 选择器：选中哪些元素——这里是所有 <h1> */
  color: red;     /* 一条声明：属性 color 设为 red */
  font-size: 32px;/* 又一条声明：字号 32 像素 */
}                 /* 一对花括号 { } 把这组声明包起来 */
```

拆解：
- `h1` —— **选择器（selector）**，决定"这组样式作用在谁身上"（第二节详解）。
- `{ ... }` —— 花括号，里面是**一堆声明**。
- `color: red;` —— 一条**声明（declaration）**，格式永远是 `属性名: 值;`，末尾分号 `;` 分隔。

### 5. CSS 怎么"接"到 HTML 上（三种挂载方式）

写好的样式必须和 HTML **连起来**才生效，有三种接法：

```html
<!-- ① 行内(inline)：直接写在元素的 style 属性里。只影响这一个元素，少用 -->
<h1 style="color: red;">标题</h1>

<!-- ② 内部(internal)：写在 <head> 的 <style> 标签里，作用于本页 -->
<style>
  h1 { color: red; }
</style>

<!-- ③ 外部(external)：单独写一个 style.css 文件，用 <link> 引进来（最推荐，可复用） -->
<link rel="stylesheet" href="style.css">
```

> 实际项目几乎都用 ③：HTML 一个文件、CSS 一个文件，`<link>` 把它们连起来。就像你的场景引用外部材质文件，而不是把材质参数抄在每个物体上。

**至此你已经能看懂 CSS 操作的对象了**：元素（盒子）、它们的 class 牌子、以及要设的样式属性。下面正式进 CSS 的四大机制。

## 一、盒模型：一切都是盒子

浏览器把每个元素都当成一个**矩形盒子**，由内到外四层：

```
┌─────────────── margin（外边距，盒子之间的距离）
│  ┌──────────── border（边框）
│  │  ┌───────── padding（内边距，内容到边框的距离）
│  │  │  content（内容：文字、图片）
```

```css
.box {
  width: 200px;      /* 内容宽度 */
  padding: 16px;     /* 内边距 */
  border: 2px solid; /* 边框 */
  margin: 8px;       /* 外边距 */
}
```
（`.box` 是"选中 class 为 box 的元素"，`.` 的含义下一节讲。）

**最大的坑（务必第一天就设）**：默认 `width` 只算 content，加了 padding/border 后盒子**实际更宽**（200 + 16×2 + 2×2 = 236px），布局老是"算不准"。解决办法是全局改成 `border-box`——让 `width` 包含 padding 和 border：

```css
*, *::before, *::after { box-sizing: border-box; }
/* 现在 width:200px 就是盒子最终宽度，padding 往里挤，直觉得多 */
```

> 记忆：`border-box` = 「所见即所设」。几乎所有现代项目开头都有这一行。（`*` 是"选中所有元素"，见下节。）

## 二、选择器与优先级：怎么选中要改的盒子

CSS 靠**选择器**找到元素，再给它加样式。回扣上一节：你在 HTML 里给元素贴了 `class`/`id` 牌子，CSS 就靠这些牌子来选。三种基础选择器：

```css
div        { }   /* 元素选择器：所有 <div>（直接写标签名） */
.card      { }   /* 类选择器：class="card" 的元素（最常用，前面加点 .） */
#header    { }   /* id 选择器：id="header" 的元素（前面加井号 #，唯一，少用于样式） */
```

**对照着看 HTML 就懂了**：
```html
<div class="card">          <!-- 被 .card 选中 -->
  <h2 class="title">标题</h2> <!-- 被 .title 选中 -->
</div>
```
```css
.card  { border: 1px solid #ddd; }  /* 选中上面那个 div，给它加边框 */
.title { color: #333; }             /* 选中里面那个 h2 */
```
> 记法：**HTML 里写 `class="card"`（不带点），CSS 里选它写 `.card`（带点）**。点号就是"按 class 找"的意思；井号 `#` 是"按 id 找"。

组合方式：
```css
.card .title    { }  /* 后代：.card 里面的 .title（中间空格） */
.card > .title  { }  /* 直接子级（> 号） */
.btn.primary    { }  /* 同时有 btn 和 primary 两个 class（无空格连写） */
.btn:hover      { }  /* 伪类：鼠标悬停在 .btn 上时才生效 */
```

**优先级（specificity）——冲突时谁赢**：同一个属性被多条规则设置，按"权重"决定。权重从高到低：

| 谁 | 权重 | 例 |
| --- | --- | --- |
| `!important` | 最高（尽量别用） | `color: red !important;` |
| 内联 style | 高 | `<div style="...">` |
| id 选择器 | 中 | `#header` |
| 类 / 伪类 / 属性 | 低 | `.card`、`:hover`、`[type=text]` |
| 元素选择器 | 最低 | `div`、`h1` |

- 权重相同时，**后写的覆盖先写的**（跟代码顺序有关）。
- 实战建议：**优先用类选择器**，别靠 id 和 `!important` 打架，否则样式很快变成一团"谁也覆盖不了谁"。这也呼应 [[01 HTML与CSS]] 讲的样式架构（BEM/CSS Modules）为什么要做隔离。

## 三、盒子怎么排：常规流与 display

不写任何布局属性时，盒子按**常规文档流（normal flow）**从上到下、从左到右堆叠。核心是 `display` 决定盒子的"排列性格"：

| `display` | 行为 | 典型标签 |
| --- | --- | --- |
| `block` | **独占一行**，可设宽高 | `<div> <p> <h1>` |
| `inline` | **同行排列**，宽高无效，只占内容大小 | `<span> <a>` |
| `inline-block` | 同行排列，但**可设宽高** | 按钮、图标 |
| `none` | 从渲染里**完全移除**（不占位） | 隐藏元素 |
| `flex` / `grid` | 开启弹性/网格布局（见 [[01 HTML与CSS]]） | 布局容器 |

```css
.tag { display: inline-block; width: 60px; }  /* 让 span 能设宽度 */
.hidden { display: none; }                     /* 彻底隐藏 */
```

> 类比：`block` 像"换行的段落"，`inline` 像"句子里的词"。这也解释了上一节为什么 `<div>` 是"块级容器"（独占一行、天生适合当大盒子）、`<span>` 是"行内容器"（跟文字挤在一行）。这套"常规流"是默认排版，Flex/Grid 是你**主动接管**排版时才开启的。

---

## 四、定位 position：脱离常规流

`position` 让盒子不再乖乖按流排，而是**精确摆放**：

| `position` | 含义 | 参照物 |
| --- | --- | --- |
| `static` | 默认，按常规流 | — |
| `relative` | 相对**自己原位**偏移，仍占原位 | 自身 |
| `absolute` | **脱离流**，绝对定位 | 最近的**非 static 祖先** |
| `fixed` | 脱离流，钉在**视口** | 浏览器窗口（滚动不动） |
| `sticky` | 滚动到阈值前是 relative，之后"粘住" | 滚动容器 |

配合 `top/right/bottom/left` 指定位置，`z-index` 控制层叠顺序（谁盖在上面）：

```css
/* 经典：父相对、子绝对——把徽标钉在卡片右上角 */
.card   { position: relative; }
.badge  { position: absolute; top: 8px; right: 8px; }

/* 导航栏吸顶 */
.header { position: sticky; top: 0; z-index: 10; }
```

> 高频坑：`absolute` 找的是"最近的**非 static 祖先**"。忘了给父级设 `position: relative`，子元素就会跑到更外层甚至整个页面去定位。

## 五、单位：别到处写死 px

| 单位 | 含义 | 用在哪 |
| --- | --- | --- |
| `px` | 绝对像素 | 边框、细节固定尺寸 |
| `%` | 相对**父元素** | 流式宽度 |
| `rem` | 相对**根字号**（`html` 的 font-size，默认 16px） | 字号、间距（推荐） |
| `em` | 相对**当前元素字号** | 跟随字号缩放的内边距 |
| `vw` / `vh` | 视口宽 / 高的 1% | 全屏区块、响应式大标题 |

```css
html { font-size: 16px; }
h1   { font-size: 2rem; }      /* = 32px，改根字号可整体缩放 */
.hero{ height: 100vh; }        /* 占满一屏高 */
```

- **经验法则**：字号和间距用 `rem`（方便整体缩放和无障碍），布局宽度用 `%` / `fr` / `flex`（见 [[01 HTML与CSS]] 响应式），只有真正固定的细节用 `px`。

## 速记

- **黑话地基**：标签`<h1>`=写法，元素=它构成的东西；`<div>`=无含义的空盒子容器（≈空 GameObject）；`class`=可复用的牌子（≈Tag/材质），`id`=唯一编号；样式=元素的视觉属性（≈材质+Transform）；CSS=写样式的语言，靠 `<link>`/`<style>`/`style` 三种方式挂到 HTML 上。
- 一切都是盒子：content → padding → border → margin。开头必设 `box-sizing: border-box`。
- 选择器：`div`（元素）/ `.类`（点）/ `#id`（井号）；HTML 里写 `class="x"`，CSS 里选写 `.x`。优先级 id > 类 > 元素，冲突看权重+顺序，优先用类。
- 常规流靠 `display`：block 独占行、inline 同行、none 隐藏；Flex/Grid 是主动接管排版。
- `position`：relative 占位偏移、absolute 找非 static 祖先、fixed 钉视口、sticky 吸顶。
- 单位：字号/间距用 rem，宽度用 %/flex，固定细节用 px。
