# 00 前端是什么

> 给「有编程基础、但没做过前端」的你。你会 Rust / C# / GLSL，这篇帮你把已有知识**迁移**到前端，不重复讲编程本身。
> 这是 [[前端知识地图]] 的起点，看完再进 [[01 HTML与CSS]]。

## 一、一句话定位

**前端 = 在浏览器里跑的程序**。你之前写的是「跑在操作系统/GPU 上的原生程序」，前端是「跑在浏览器这个运行时里的程序」。

浏览器就像 Unity 引擎：它提供了运行时（JS 引擎）、渲染系统（DOM/CSS）、资源加载、事件系统，你写代码去驱动它。

## 二、三大支柱（各管一件事）

前端页面由三种语言协作，职责分明：

| 语言 | 类比 | 管什么 |
| --- | --- | --- |
| **HTML** | 场景里的物体层级（Hierarchy） | 页面有哪些**内容/结构**（标题、按钮、图片） |
| **CSS** | 材质 / Shader | 这些内容**长什么样**（颜色、布局、大小、动画） |
| **JavaScript** | C# 脚本 / 游戏逻辑 | 页面的**行为**（点击、请求数据、动态更新） |

- 详见 [[01 HTML与CSS]]（结构+样式）、[[02 JavaScript核心]]（行为逻辑）。
- **TypeScript** = 给 JavaScript 加了静态类型的版本，就像从「动态语言」升级到「有类型检查」。你有 Rust/C# 背景会非常适应，见 [[03 TypeScript]]。公司项目基本都用 TS。

## 三、一个页面是怎么跑起来的

从你熟悉的角度理解这条链路（细节见 [[04 浏览器原理]]）：

```
用户输入网址
  → 浏览器下载 HTML（相当于加载场景描述）
  → 解析 HTML 建立 DOM 树（相当于 Scene 的节点树 / 你做的 Node3D 树）
  → 解析 CSS，算出每个节点长什么样
  → 排版(Layout) + 绘制(Paint) + 合成(Composite) 到屏幕
  → 执行 JS，绑定事件、动态修改 DOM
```

**DOM（Document Object Model）**：页面结构在内存里的树状表示。JS 通过操作 DOM 来改变页面——这跟你在 [[Tripo Orbit]] 里操作场景节点树、标脏重绘是**同一种思想**（见 [[08 Web3D与AI]] 的脏节点）。

**关键差异（务必记住）**：
- 浏览器里的 JS 是**单线程**的（见 [[04 浏览器原理]] 事件循环）。不像 Rust 可以随便开线程，前端重计算要靠 **Web Worker** 或拆分任务，否则界面会卡死。
- 前端没有 `main()` 入口那种「一次跑完」，而是**事件驱动**：加载完就等着，用户点一下 / 数据回来了，才触发对应回调。类似 Unity 的 `Update` / 事件回调模型。

## 四、你的业务在前端的哪个位置

结合 [[任务需求]] 里你做的方向：
- **Web 端 shader**（ts/js + WebGL + GLSL）：这是「前端 + 图形」的交叉。前端负责搭页面、拿到 canvas、传数据；GLSL 那部分你已经会了（见 [[08 Web3D与AI]] 的模糊 shader、描边）。
- **DCC Bridge 的网页侧**：网页把模型发给 Unity 插件，涉及 WebSocket、数据传输，属于前端的「网络通信」部分（见 [[05 计算机网络]]）。
- 和同事对接时，他们说的「组件、状态、路由、打包、Vue/React」这些词，就是接下来这套文档要补的。

## 五、学习建议（针对你）

1. **别纠结「背语法」**：你会编程，重点是理解「浏览器这个运行时的规则」和「生态工具」。
2. **顺序**：先看 [[00.1 环境与工具链]] 把项目跑起来 → [[01 HTML与CSS]] → [[02 JavaScript核心]] → [[03 TypeScript]]，前四块是根基。
3. **框架（Vue/React）不急**：先懂原生三件套，再看 [[06 Vue与React]]，否则容易「只会框架不懂原理」。
4. **和同事对接的协作流程**看 [[00.2 与同事对接]]。

## 速记

- 前端 = 跑在浏览器里的程序，浏览器 ≈ 运行时/引擎。
- HTML 管结构、CSS 管样式、JS 管行为、TS 给 JS 加类型。
- DOM = 页面的节点树，JS 改 DOM 来更新画面。
- 单线程 + 事件驱动，是它和原生开发最大的不同。
