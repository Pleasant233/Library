## 一、语法速览（迁移，快过）

大部分能力你已具备，只列**和 C++/C#/Python 不同**或**必须知道**的：

```js
let x = 1          // 可变（≈ C# 的普通变量）
const y = 2        // 不可变绑定（≈ C# 的 readonly / C++ 的 const）——默认优先用 const
// 没有 var（老语法，有作用域坑，别用）

// 动态弱类型：变量能随时换类型（≈ Python 的动态类型；类型安全交给 TS，见 [[03 TypeScript]]）
let v = 1; v = 'now string'   // 合法但危险

`模板字符串：${x + y}`         // 反引号插值（≈ Python 的 f-string / C# 的 $"...")

const add = (a, b) => a + b   // 箭头函数（≈ C# lambda (a,b)=>a+b / Python lambda）

const { name, age } = user    // 解构（≈ Python 的解包 name, age = ...）
const arr2 = [...arr, 4]      // 展开（≈ Python 的 [*arr, 4]）

// 数组是主力，方法链式风格（≈ C# 的 LINQ / Python 列表推导）
;[1,2,3].map(n => n*2).filter(n => n>2)   // [4, 6]

// 对象字面量就是键值表，没有 struct/class 定义也能直接用（≈ Python dict）
const p = { x: 1, y: 2 }
```

**两个相等坑**：`==` 会隐式转类型（`0 == '0'` 为 true），**永远用 `===`**（严格相等，不转类型）。判空常用 `??`（`a ?? b`：a 为 null/undefined 才取 b）和 `?.`（`a?.b`：a 不存在就返回 undefined 而非报错）。

## 二、DOM 操作：JS 怎么改页面

这是前端的**核心动作**，也是你全新的部分。[[00 前端是什么]] 说过：DOM 是页面在内存里的节点树，改 DOM = 改画面。基本套路「**选中 → 改 / 建 / 删 → 绑事件**」：

```js
// 1. 选中（CSS 选择器语法，呼应 [[00.3 CSS上手]]）
const btn = document.querySelector('.btn')       // 第一个匹配
const items = document.querySelectorAll('li')    // 所有匹配（类数组）

// 2. 改内容 / 属性 / 样式 / 类
el.textContent = '新文字'          // 改文本（安全）
el.innerHTML = '<b>富文本</b>'     // 改 HTML（注意 XSS，见 [[04 浏览器原理]]）
el.setAttribute('href', '/x')     // 改属性
el.style.color = 'red'            // 改内联样式
el.classList.add('active')        // 加/删/切类（推荐用类控制样式，别堆 style）
el.classList.toggle('open')

// 3. 建 / 插 / 删
const li = document.createElement('li')
li.textContent = '新项'
list.appendChild(li)              // 插入
li.remove()                       // 删除

// 4. 绑事件（前端的「入口」——事件驱动，见 [[00 前端是什么]]）
btn.addEventListener('click', (e) => {
  console.log('被点了', e.target)
})
```

> 关键心智：前端没有 `main()` 跑一遍就结束，而是**绑好事件后等着**，用户操作/数据回来才触发回调。框架（Vue/React）的意义就是让你**不用手写这些 DOM 操作**，改成「改数据、视图自动更新」（见 [[06 Vue与React]]）——但先懂手动版，才懂框架帮你省了什么。

## 三、`this` 与原型：JS 最容易踩的坑

**`this` 是动态的**——它的值取决于**函数怎么被调用**，不是定义在哪（和 C# 的 `this` 完全不同，这是头号坑）：

```js
const obj = {
  name: 'A',
  hi() { console.log(this.name) },
}
obj.hi()                    // 'A'（this = obj，谁调用指向谁）
const f = obj.hi
f()                         // undefined（脱离 obj 调用，this 丢了）

// 箭头函数不绑自己的 this，而是「继承定义处」的 this —— 用它避坑
setTimeout(() => console.log(this.name), 0)   // this 还是外层的
```

**原型（prototype）**——JS 的「继承」机制：对象顺着 `__proto__` 链向上找属性/方法。日常用 `class` 语法糖即可，它底层就是原型：

```js
class Animal {
  constructor(name) { this.name = name }
  speak() { return `${this.name} 叫` }
}
class Dog extends Animal {          // 继承
  speak() { return `${this.name} 汪` }  // 重写
}
new Dog('旺财').speak()             // '旺财 汪'
```

> 够用结论：写业务用 `class` 和箭头函数，就能绕开大部分 `this`/原型坑。深入原理（原型链、闭包如何持有作用域）见 [[02 JavaScript核心]]。

## 四、异步：Promise / async-await / fetch

JS 单线程（见 [[00 前端是什么]]），**耗时操作（网络、定时器）不能阻塞**，全靠异步。演进三代：

```js
// 1) 回调（老，易「回调地狱」层层嵌套）
setTimeout(() => { /* 1 秒后 */ }, 1000)

// 2) Promise：代表「未来会有的值」，链式 .then
fetch('/api/user')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err))

// 3) async/await：把异步写得像同步（现在主流，最易读）
async function loadUser() {
  try {
    const res = await fetch('/api/user')   // 等请求回来
    const data = await res.json()          // 等解析
    return data
  } catch (err) {
    console.error('请求失败', err)
  }
}
```

- **`fetch`** 是浏览器内置的发请求 API（≈ 你熟悉的 HTTP client），返回 Promise。细节（状态码、跨域、WebSocket）见 [[05 计算机网络]] 和 [[10 前端接WebSocket]]。
- **`await` 只能在 `async` 函数里用**。`await` 期间不阻塞主线程，浏览器照样响应用户。
- 这套异步「什么时候轮到谁跑」的精确规则，就是 [[02 JavaScript核心]] 的**事件循环**——那篇建立在这里之上。

## 五、模块：import / export

代码拆成多个文件，用 **ES Module（ESM）** 互相引用（≈ Python 的 `import` / C# 的 `using` + 命名空间）。这也是 [[00.1 环境与工具链]] 里打包工具处理的对象：

```js
// utils.js —— 导出
export function formatDate(d) { /* ... */ }
export const PI = 3.14
export default class Foo {}      // 默认导出（一个文件一个）

// main.js —— 导入
import Foo, { formatDate, PI } from './utils.js'
import { ref } from 'vue'         // 从第三方包（node_modules）导入
```

- **具名导出**用 `{}` 导入、名字要对上；**默认导出**导入时可任意起名。
- ESM 是**静态**的（编译期能分析出谁没被用），这正是 [[02 JavaScript核心]] **Tree Shaking** 能摇掉无用代码的前提。

## 速记

- 语法：`const` 优先、`===` 别用 `==`、`?.` / `??` 防空、箭头函数 + 解构 + 数组方法链。
- DOM：`querySelector` 选中 → 改 `textContent`/`classList`/建删节点 → `addEventListener` 绑事件。事件驱动，绑完就等。
- `this` 看**怎么调用**不看定义处；用 `class` + 箭头函数避坑；原型是底层继承机制。
- 异步：回调 → Promise → `async/await`；`fetch` 发请求；精确调度看事件循环。
- 模块：`import/export`（ESM 静态可摇树）。
