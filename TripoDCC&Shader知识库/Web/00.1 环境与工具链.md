# 00.1 环境与工具链

> 前端的「工具链」是新人最容易懵的地方——一堆名词。这篇把它们的**职责**讲清，让你能把项目跑起来、看懂同事的配置。
> 类比：这相当于前端世界的 `cargo` / `rustup` / `Cargo.toml`。

## 一、运行时：Node.js

- **Node.js** = 把浏览器的 JS 引擎（V8）搬到操作系统上，让 JS 能在**命令行/服务器**跑，能读写文件、跑构建工具。
- 类比：Node 之于前端工具链，约等于「让 JS 脱离浏览器也能运行的环境」。
- 你几乎所有前端命令（安装依赖、启动项目、打包）都跑在 Node 上。
- **版本管理**：用 `nvm`（类似 Rust 的 `rustup`）管理多个 Node 版本。项目常有 `.nvmrc` 指定版本。

## 二、包管理器：npm / pnpm / yarn

对应 Rust 的 `cargo` + crates.io：

| 工具 | 说明 |
| --- | --- |
| **npm** | Node 自带，最基础 |
| **pnpm** | 更快、省磁盘（软链接共享依赖），大厂常用 |
| **yarn** | 另一个流行选择 |

- **`package.json`** ≈ `Cargo.toml`：声明项目信息、依赖、可执行脚本。
- **`node_modules/`** ≈ `target/` + 下载的 crates：装依赖的地方，巨大，**不提交 git**。
- **`package-lock.json` / `pnpm-lock.yaml`** ≈ `Cargo.lock`：锁定精确版本，保证团队装的一样。**要提交**。

常用命令：
```bash
pnpm install          # 装依赖（等于 cargo build 拉依赖），简写 pnpm i
pnpm add axios        # 加一个依赖（等于 cargo add）
pnpm dev              # 启动开发服务器（跑 package.json 里 scripts.dev）
pnpm build            # 打包生产版本
```

> `scripts` 字段里定义的命令用 `pnpm <脚本名>` 运行。第一次接手项目：先看 `package.json` 的 `scripts` 有哪些。

## 三、为什么需要「构建/打包」

浏览器原本只认 HTML/CSS/JS，但你写的是 TS、Vue 组件、还 import 一堆模块——这些浏览器**不直接认识**，需要工具**转译 + 打包**成浏览器能跑的东西。

这一步类比你熟悉的**编译**：源码（TS/Vue）→ 编译打包 → 浏览器可运行的产物（bundle）。

**Vite**（现在主流，你的项目大概率用它）：
- **开发时**：秒级启动，改代码浏览器**热更新（HMR）**，不用刷新——类似热重载。
- **打包时**：把所有代码优化、压缩、**Tree Shaking**（摇掉没用的代码，见 [[02 JavaScript核心]]）成小体积产物。
- 老一点的项目可能用 **Webpack**，慢一些，概念类似。

## 四、类型与代码质量工具

| 工具 | 作用 | 类比 |
| --- | --- | --- |
| **TypeScript (tsc)** | 类型检查 | Rust 的类型系统/编译器检查 |
| **ESLint** | 查代码问题、坏味道 | `cargo clippy` |
| **Prettier** | 自动格式化 | `cargo fmt` / `rustfmt` |

- 这些通常在**提交代码前**自动跑（见 [[00.2 与同事对接]] 的 Git Hooks），不符合规范会拦住你。别关掉它们，团队靠它保持代码一致。

## 五、一个典型前端项目长什么样

```
project/
├── package.json          # 依赖 + 脚本（先看这个）
├── pnpm-lock.yaml         # 锁版本
├── vite.config.ts         # 构建配置
├── tsconfig.json          # TS 配置
├── .eslintrc / .prettierrc# 代码规范配置
├── index.html             # 入口 HTML
├── src/                   # 你的源码在这
│   ├── main.ts            # JS 入口
│   ├── components/        # 组件
│   └── assets/            # 图片等静态资源
└── node_modules/          # 依赖（不提交）
```

## 六、接手一个陌生项目的步骤

1. 装对 Node 版本（看 `.nvmrc`）：`nvm use`
2. 装依赖：`pnpm install`
3. 看 `package.json` 的 `scripts`，找启动命令：通常 `pnpm dev`
4. 打开它提示的本地地址（如 `http://localhost:5173`）
5. 改 `src/` 里的代码，浏览器自动热更新
6. 看不懂配置不要慌，先跑起来，用到再查

## 速记

- Node = JS 的运行时（≈ 让 JS 能跑在命令行）。
- pnpm/npm = 包管理器（≈ cargo），`package.json` ≈ `Cargo.toml`。
- Vite = 构建工具，开发热更新 + 打包压缩（≈ 编译）。
- ESLint ≈ clippy，Prettier ≈ rustfmt，TS ≈ 类型检查。
- 接手项目：`nvm use` → `pnpm i` → `pnpm dev`。
